# BSA16 — Интеграция веб-сервисами SOAP

## Описание проекта
Проект посвящён интеграции ИТ-систем с использованием протокола SOAP. В рамках проекта изучены основы XML/XSD/WSDL, разобраны структуры SOAP-сервисов и выполнено обратное проектирование требований по WSDL-документам для двух сервисов: управление участниками клуба и их питомцами.

---

## Что было сделано

**ex00_wsdl_structure.xlsx**  
Определены логические части WSDL (теги первого уровня) и описано назначение каждой части.

**ex01_porttype_methods.xlsx**  
Выделены операции (методы) сервисов из раздела `<portType>` и описано их назначение.

**ex02_schema_elements.xlsx**  
Разобраны элементы раздела `<schema>`: назначение `element`, `complexType`, `sequence`, а также атрибуты `type`, `minOccurs`, `maxOccurs` и правила обязательности.

**ex03_reverse_engineering_messages.xlsx**  
Выполнено обратное проектирование по WSDL: описаны форматы входных и выходных сообщений для операций `addMember`, `addPet`, `updatePet`, `getAllMembers` (типы, массивность, обязательность, примечания).

**ex04_reverse_engineering_notes.docx**  
Сформулировано отличие выходных сообщений операций `getAllMembers` и `getMemberByName`.

**ex05_endpoints.xlsx**  
Определены endpoint (URL) для SOAP-сервисов `member` и `pet` по WSDL-документам.

---

## Итоги
Проект позволил понять принципы SOAP-интеграции и научиться читать формальные контракты веб-сервисов. Я освоил анализ WSDL/XSD, определение операций и endpoint, а также обратное проектирование требований и форматов сообщений, что помогает формализовать интеграционные требования и готовить спецификации для разработки.
