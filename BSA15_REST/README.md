# BSA15 — Интеграция веб-сервисами REST

## Описание проекта
Проект посвящён интеграции ИТ-систем с использованием архитектурного стиля REST. В рамках проекта проработано интеграционное взаимодействие для системы онлайн-записи в барбершоп: актуализированы Use Case, описаны REST-методы и структуры сообщений, выполнен маппинг данных и сформированы требования к обработке ошибок.

---

## Что было сделано

**ex00_terms.xlsx**  
Сопоставлены ключевые термины REST и их определения (REST, кеширование, масштабируемость, асинхронность).

**ex01_data_preparation.docx / .xlsx**  
Подготовлены исходные данные из предыдущих проектов: классы и атрибуты, необходимые для UC02 и UC04, включая варианты из другой группы.

**ex02_uc04_update.docx**  
Актуализирован UC04 «Зарегистрировать клиента (менеджер)»: детализированы шаги и альтернативы, указаны роли и приложения, отмечены интеграционные вызовы и справочники.

**ex03_reference_integration.xlsx**  
Проработана интеграция справочников, используемых в UC04: определена потребность интеграции и выбран вариант (полный справочник / дельта / связанные записи / поиск по мере необходимости) с указанием методов.

**ex04_rest_methods_uc04.docx**  
Определены REST-методы для интеграции UC04: выбран HTTP-метод, сформирован URL-путь, описаны параметры и тело запроса, подготовлены варианты успешных и ошибочных ответов.

**ex05_request_response_schema.xlsx**  
Описаны элементы тела запросов и ответов: блоки и поля, типы, размерность, обязательность, комментарии и ссылки на справочники.

**ex06_mapping.xlsx**  
Выполнен маппинг данных: сопоставление полей между системой-отправителем, сообщением и системой-получателем с учетом типов, обязательности и ограничений.

**ex07_errors.xlsx**  
Описаны требования к обработке ошибок интеграции: проверки полей, коды и сообщения ошибок, критичность, обработка недоступности внешней системы и проверка ссылок на справочники.

## Итоги
Проект позволил освоить подготовку REST-интеграции на уровне аналитика: от актуализации Use Case и выбора методов до описания контрактов запрос/ответ, маппинга данных и требований к обработке ошибок. Я научился формировать интеграционную документацию, пригодную для разработки и интеграционного тестирования.
