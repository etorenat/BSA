# BSA14 — Интеграция ИТ-систем

## Описание проекта
Проект посвящён анализу интеграции ИТ-систем и работе системного аналитика при проектировании обмена данными. В рамках проекта для задачи доставки заказов выявлены потоки данных, определены мастер-системы и системы-получатели, разработан словарь данных и описаны сценарии интеграции, а также сформулированы требования к объёмам, частоте и срокам передачи данных.

---

## Что было сделано

**ex00_data_flows.xlsx**  
Выявлены и описаны потоки данных для системы доставки: название, назначение и направление (входящий/исходящий).

**ex01_master_systems.xlsx**  
Для входящих потоков определены возможные источники и выбраны мастер-системы с обоснованием.

**ex02_receiver_systems.xlsx**  
Для исходящих потоков определены системы-получатели, проверена согласованность мастер-систем и получателей для связанных данных.

**ex03_data_dictionary.xlsx**  
Разработан словарь данных по каждому потоку: структура, вложенность, обязательность блоков, атрибуты (тип, длина, ограничения).

**ex04_integration_scenarios.docx / .png**  
Описаны сценарии интеграции для потоков данных в виде Use Case и/или диаграмм.

**ex05_volume_requirements.xlsx**  
Определены требования к объёмам и количеству данных: средние и пиковые значения, периоды нагрузки, размеры сообщений.

**ex06_frequency_sla.xlsx**  
Определены требования к частоте и срокам передачи: желаемые и допустимые диапазоны частоты и времени доставки.

---

## Итоги
Проект позволил системно подойти к интеграции данных между ИТ-системами: от выявления потоков и участников обмена до описания структуры данных и требований к нагрузке и SLA. Я научился определять мастер-системы и получателей, формировать словарь данных и описывать сценарии интеграции, что является основой для выбора интеграционного решения и дальнейшей проработки маппинга и ошибок обмена.
